Softwares de Versionamento

Sistema que tem por objetivo realizar o CONTROLE DE VERSÕES DE UM CÓDIGO ou DOCUMENTO.

É possível gerenciar as versões de um código ou documento realizadas por um desenvolvedor ou equipe de desenvolvedores.

Software de controle de versões

Cria um repositório local e a partir do arquivo cria um histórico de versões.

  • O arquivo é versionado, ou seja, cada alteração em cada versão.
Além do repositório local, é possível criar um repositório remoto (na nuvem)
Assim mais pessoas pessoas podem alterar com mais controle e segurança

Tipos de Software de controle de versão

Centralizado

Possui um repositório central em um servidor com controle de versões (depende do servidor)

Ex: Software subversion

Distribuído

Possui um repositório local com controle de versões

  • Atualizações no repositório central
  • Independente do servidor

Softwares mercurial, Git

GIT

É um software livre de versionamento que surgiu para o desenvolvimento do Kernel do Linux,
foi projetado e desenvolvido por LINUS TORVALDS em 2005.

Cada diretório de trabalho do GIT é um repositório com um histórico completo e habilidade total
de acompanhamento das revisões, não depende de acesso a uma rede ou servidor central.

GITHUB

Plataforma de hospedagem de código-fonte e arquivos com controle de versão.

Surgiu em 2008 e foi adquirido pela MICROSOFT em 2018 por 7,5 bilhões de dólares.

Hospeda cerca de 100 milhões de projetos e tem 50 milhões de usuários

OCTOCAT

Mascote do GITHUB

  • Tem vários braços (Branchs)
  • Junção de polvo com gato (Merge)

Para realizar o versionamento dos projetos é necessário:

  1. GITHUB
  2. GIT
  3. Desktop GITHUB
  4. Visual Studio Code

Procedimento:

  1. Baixar o GIT
  2. Criar conta no GITHUB
  3. Baixar o GITHUB Desktop
  4. VS Code

COMANDOS GIT

  • Commit- torna permanente um conjunto de alterações em um repositório
  • Push- envia alterações para o servidor no repositório remoto
  • Fetch- verifica se o seu repositório local está em sincronia com o remoto
  • Pull- sincroniza o repositório local com o remoto atualizando as versões

O repositório remoto possibilita que as alterações sejam feitas em diferentes máquinas.
